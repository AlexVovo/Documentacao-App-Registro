7.2.1 Arquitetura do Projeto

O projeto segue um padrão arquitetural baseado em MVC (Model-View-Controller), promovendo organização, separação de responsabilidades e facilidade na manutenção do código.

📁 Estrutura de Pastas

lib/
├── model/                # Modelos de dados (ex: usuário, paciente, entrevista)
├── pagesapp/             # Telas da aplicação (interface do usuário)
├── service/              # Serviços como Firebase, APIs externas e geração de PDF
├── firebase_options.dart # Configuração do Firebase (gerado automaticamente)
└── main.dart             # Ponto de entrada da aplicação

🧩 Descrição das Camadas

📁 model/
Contém as classes de dados utilizadas no aplicativo. São representações das entidades que trafegam entre as camadas da aplicação.
Exemplos: UserModel, PacienteModel, EntrevistaModel.

📁 pagesapp/
Esta pasta corresponde à camada de View. Nela estão implementadas todas as telas do app, construídas com widgets Flutter.

📁 service/
Responsável por implementar os serviços de backend e funcionalidades auxiliares, como:

  • Acesso ao Firebase (Auth e Firestore)
  • Comunicação com APIs REST externas
  • Geração de relatórios em PDF

📄 firebase_options.dart
Arquivo gerado automaticamente com as configurações do projeto Firebase. É utilizado na inicialização do Firebase no main.dart.

📄 main.dart
Ponto de entrada da aplicação Flutter. Responsável por:

  • Inicializar o Firebase
  • Definir a tela inicial (home)
  • Configurar o tema global do app

🔄 Fluxo Arquitetural

Usuário → pagesapp/ (View)
                ↓
         service/ (Serviços)
                ↓
         model/ (Dados)
